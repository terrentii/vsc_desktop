У меня есть идея проекта: МЫС Desktop, это приложение анонимный мессенджер для компьютеров, у меня уже есть МЫС Web, который рабоает на сервере уентрализовано вот гитхаб - https://github.com/terrentii/vsc_web, он уже задеплоен на сервер soufos.ru. 
Что я хочу от приложения? 
У приложения есть 2 режима: Централизованный и Децентрализованный. Что и является осноной функцией, вот описание двух этих режимов:
Децентрализованный:
Мессенджер работает по ssh, то есть подключается к другим компьютерам и передает им сообщения по shh без центрального сервера, как компьютеры с серыми IP будут друг друга находить? Это хороший вопрос, один из моих вариантов - У меня же есть сервер, можно линковаться через него, а потом работать уже без него.
Ещё раз всё вместе: в децентирализованном режим пользователь должен просто открыть приложение ввести какой-либо ключ, и его подключит к другому пользователю по ssh и они смогут общаться по защищённому каналу, работает без аккаунта или чего-нибудь такого, упор на анонимность - данные должны храниться на компьютере пользователей, возможно в файле (он обязательно должен быть зашифрован), также я бы наметил путь для отправки сообщений. Очень важно - отдельно нужно будет создать новый протокол шифровки, достаточно мощный и подходящий по моим критериям.
Централизованный:
Мессенджер работает через сервер, который у меня есть, то есть пользователь входит в свой аккаунт МЫС на компьютере, после чего, содержимое веб версии приложения синхронизируется с десктопом, то есть пользователь может писать сообщения, переключаться между группами, в общем всё как на сайте, только через приложение.

Проект должен выйти идеально анонимным и безопасным - упор делать на этом. То есть своё шифрование (например, через несколько мат. функций), ssh, и т.д
