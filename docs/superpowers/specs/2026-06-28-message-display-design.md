# Спека: отображение сообщений (под-проект A)

Дата: 2026-06-28. Часть декомпозиции «медиа везде» на A → B → C:

- **A (эта спека)** — редизайн отображения сообщений как в `vsc_web`: фундамент
  данных (автор, серверное время, колонка медиа), «кто отправил», блоки кода,
  рендер медиа. Без транспорта файлов.
- **B** — медиа в режиме «Центр» (загрузка/скачивание через сервер). Зависит от A.
- **C** — медиа в P2P (E2E-передача файлов). Зависит от A.

Эталон дизайна — `vsc_web`: `templates/room.html`, CSS в `templates/base.html`
(`.msg`, `.msg-author{,.is-me,.is-anon,.is-founder}`, `.code-block`, `.msg-media`),
серверный фильтр `render_text` (`app.py:184`), клиентский ре-хайлайт на highlight.js.

## Постановка (что не так сейчас)

Новый «журнальный» UI (ветка дизайна, влита в master) показывает имя автора и
время, но данных под ними нет — централизованный синк их роняет:

1. **Имена.** Сервер шлёт `sender` (`mys_centralized/models.py:37`), синк его
   получает (`sync.py:101`), но в `messages` нет колонки автора
   (`mys_storage/schema.py:35–43`), и `repositories.add` его не принимает
   (`repositories.py:114`). UI для всех входящих показывает запасной `peer_label`.
2. **Время.** Серверный `created_at` (`models.py:39`, `sync.py:103`) не
   сохраняется; `repositories.add` штампует всем входящим `received_at =
   time.time()` — момент синхронизации (`repositories.py:116,122`). При загрузке
   истории все сообщения получают одинаковое «сейчас».
3. **Медиа.** Нет ни колонки, ни рендера. В вебе есть `msg.media`.

Перезаход не помогает — данные не сохраняются в принципе. Это не регрессия мёрджа,
а вскрывшийся пробор: старый минималистичный UI просто не показывал имя/время.

## Объём A

В A: фундамент данных (`author`, `created_ts`, `media`), проводка `sender`/
`created_at` через синк, редизайн ленты (журнал как в вебе), различение
отправителя цветом имени, блоки кода с копированием, рендер медиа по локальной
ссылке (на синтетических данных — реальную доставку дают B/C).

Не в A: загрузка/скачивание/E2E-передача файлов (это B и C); подсветка синтаксиса
кода; бейдж «ПРАРОДИТЕЛЬ» (десктоп не знает `creator_login`).

## A1. Фундамент данных

Миграция схемы (следующая версия в `schema.py`), `ALTER TABLE messages`:

- `author TEXT` — отправитель. «Центр»: серверный `sender`. P2P: `NULL`
  (имя берётся из `peer_label`, диалог 1:1).
- `created_ts REAL` — серверное время (epoch из распарсенного `created_at`).
  Локальные `sent_at`/`received_at` не трогаем.
- `media TEXT` — необязательная локальная ссылка на вложение; в A только колонка
  и рендер, заполняют B/C.

`MessagesRepo.add(...)` получает новые необязательные параметры `author=None`,
`created_ts=None`, `media=None` и пишет их в INSERT. Существующие вызовы (P2P,
локальные исходящие) работают без изменений (значения по умолчанию `NULL`).

`SyncEngine._ingest`/`ingest_ws` (`sync.py`): при `messages.add(direction="in",
...)` передают `author=msg.sender` и `created_ts=_parse_iso(msg.created_at)`.
Хелпер `_parse_iso` парсит ISO-таймстамп сервера в epoch (UTC); при ошибке —
`None` (тогда UI откатывается на `received_at`, как сейчас).

## A2. Архитектура рендера

Лента остаётся `QListWidget` (UI-тесты опираются на `count()`, `item(i)`,
`item(i).text()`). Сырое тело по-прежнему в `DisplayRole`. Но вместо отрисовки
делегатом каждая строка получает реальный виджет через
`setItemWidget(item, MessageRow(...))` — это нужно для интерактива (кнопка
«Копировать», картинки), который делегат рисует плохо.

`MessageRow(QWidget)` собирает вертикально:

1. **Заголовок** — имя (цвет по роли, см. A3) + моно-время.
2. **Тело** — последовательность сегментов из парсера A4: текст → `QLabel`
   (`wordWrap`, перенос строк), код → `CodeBlock` (A4).
3. **Медиа** — `MediaView` (A5), если у сообщения есть `media`.

Высота строки — из `MessageRow.sizeHint()` (через `item.setSizeHint`). Hairline
между строками сохраняется (рамка/разделитель в стиле виджета).

`ChatView.show_messages(messages, *, peer_label)` для каждого сообщения создаёт
`QListWidgetItem(raw_body)` и ставит ему `MessageRow`. Логика выбора имени/времени
— в `ChatView` (передаёт готовые значения в `MessageRow`).

## A3. «Кто отправил»

Различение — цветом имени (как `.msg-author` в вебе), без выравнивания/пузырей:

- своё (`direction == "out"`): имя = «я», цвет `tokens()["accent"]` (кобальт);
  аватар-квадрат кобальтовый (уже есть в текущем рендере).
- чужое: имя = `author` (для «Центра») либо `peer_label` (P2P/нет автора),
  цвет `tokens()["text"]`.
- anon (имя начинается с `Anon`): mono-шрифт, приглушённый цвет (`text3`) —
  аналог `.is-anon`.
- `ponytail:` бейдж «ПРАРОДИТЕЛЬ» (`.is-founder`) отложен — нет `creator_login`;
  точка расширения, когда сервер начнёт отдавать признак основателя.

## A4. Блоки кода

Парсер `split_segments(body: str) -> list[tuple[str, str]]` режет тело по
```` ``` ````-ограждениям (как `render_text`, `app.py:187`): чётные части —
`("text", ...)`, нечётные — `("code", ...)` с обрезкой ведущего/хвостового
перевода строки. Инвариант (юнит-тест): для текста без ``` —
`split_segments(t) == [("text", t)]`, и склейка сегментов восстанавливает исходные
символы.

`CodeBlock(QWidget)` повторяет структуру оригинала: бар «//// code» (метка) +
кнопка «Копировать» (кладёт код в `QApplication.clipboard()`), ниже моноширинный
`pre` (`QLabel`/`QPlainTextEdit` только для чтения). Цвета — из текущей темы
(`tokens()`), тёмная/светлая. **Без подсветки синтаксиса** (решение по объёму).

## A5. Рендер медиа

`MediaView(ref: str, parent=None)` по локальной ссылке `media`:

- изображение (по расширению `.png/.jpg/.jpeg/.gif/.webp`): `QPixmap`,
  масштаб ≤ 320×280 с сохранением пропорций, рамка 1.5px; клик открывает файл.
- иное: плашка-ссылка с 📎 и именем файла (как `.msg-media-link`).

В A `media` — это локальный путь; тестируется на синтетическом сообщении с
`media`-путём к временному файлу. Источник реальных файлов (скачивание из «Центра»,
приём по P2P) — в B/C. Видео из вебе (`<video>`) в A не делаем — добавится с B.

## A6. Границы модулей

- `mys_storage` (миграция, `repositories.add`) — не знает об UI и сети.
- `mys_centralized.sync` — парсит серверное время и прокидывает `sender`; не знает
  о виджетах.
- `mys_ui.widgets` (`ChatView`, `MessageRow`, `CodeBlock`, `MediaView`,
  `split_segments`) — чистый UI/парсинг, данные приходят готовыми dict-ами.

## A7. Тестирование

- **Миграция** (`mys_storage`): новые колонки существуют; `add(author=..,
  created_ts=.., media=..)` сохраняет и `list()` возвращает их; старые вызовы без
  новых параметров пишут `NULL`.
- **Синк** (`mys_centralized`): `_ingest`/`ingest_ws` сохраняют `author=sender` и
  `created_ts` из распарсенного `created_at`; битый `created_at` → `created_ts None`.
- **Парсер** (`split_segments`): инвариант восстановления; смесь текст/код;
  код без закрывающего ограждения; пустой ввод.
- **UI** (headless `offscreen`): своё имя цветом `accent`, чужое — `text`;
  сообщение с ``` → строка содержит `CodeBlock`; время берётся из `created_ts`,
  при его отсутствии — из `received_at`; сообщение с `media` → есть `MediaView`.
  Сохранить совместимость существующих UI-тестов (`count()`, `item(i).text()`).
